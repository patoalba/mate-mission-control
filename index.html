Tecnologías, Directorios y Modelos de Datos

Explorá el diseño de las bases de datos de Firestore, estructura del monorepo y secrets configurados.

Modelado de Datos (Firestore Native)

Definiciones de colecciones y convenciones de seguridad por usuario (Namespacing).

Colección Principal: users/{userId}
Campos Clave:
- id: UID del usuario (Firebase/FastAPI OAuth)
- phoneE164: Número WhatsApp (ej. "+54911...") para ruteo.
- onboardingState: "started" | "completed" | "drive_indexing".
- registeredAt: timestamp de creación.
establishments / plots

Representa los campos y lotes asociados. plots contiene geometría (KML/coordenadas) e históricos de cultivo. soft-deleted en backend.

logEntries / reminders

Bitácora de campo que rellena el usuario (WhatsApp) y agenda de recordatorios disparada por Cloud Scheduler cada 5 min.

conversations/{userId}

Historial resumido de la conversación (rolling summary) de hasta 2000 caracteres. Protegido rigurosamente mediante firestore.rules.

documents/{documentId}

Metadata de archivos indexados por RAG (Drive, emails o adjuntos). Almacena el contentHash de idempotencia y el gcsPath.

Topología de Directorios del Monorepo

La separación entre `mate-infra` (IaC) y `mateai` (código de los microservicios).

📂 ~/Documents/Mate/ ├── 📄 CLAUDE.md # Convenios de desarrollo globales ├── 📂 mate-infra/ # Código de Terraform (Aplicado en producción) │ ├── 📂 terraform/ │ │ ├── 📂 envs/prod/ # Configuración activa del entorno productivo │ │ └── 📂 modules/ # Módulos Cloud Run, Firestore, PubSub, IAM, Vertex │ └── 📄 firestore.rules # Reglas de seguridad basadas en UID del usuario └── 📂 mateai/ # Monorepo de lógica de Mateo ├── 📄 pyproject.toml # Gestión de dependencias general (uv) ├── 📂 domain/ # Modelos de dominio Pydantic v2 compartidos ├── 📂 agent/ # System instruction, prompts y herramientas del LLM └── 📂 services/ ├── 📂 ingest/ # FastAPI Webhook, firma HMAC y ruteo a PubSub ├── 📂 processor/ # Invocación a Vertex AI Agent, ejecución de tools └── 📂 indexer/ # Pipeline batch (Document AI + embeddings)
Gestión de Secretos (Secret Manager)

Secrets aprovisionados y mapeados de forma segura como variables de entorno a los pods de Cloud Run sin keys físicas.

whatsapp-access-token & whatsapp-app-secret

Token permanente e identificación HMAC provistos por Meta for Developers para validar firmas webhooks de WhatsApp y responder mensajes.

jwt-signing-key

Llave criptográfica de 32+ bytes para la firma de Magic Links e inicio de sesión seguro en el dashboard de Next.js (`web` service).

mate-drive-sa-key

Llave de la cuenta de servicio con permisos delegados para conectarse a Drive API en nombre del usuario (Sincronizador RAG).
